🔥 ВАРИАНТЫ ДЛЯ DESKTOP OUTLOOK
⚡ ВАРИАНТ 1: ЦЕНТРАЛИЗОВАННЫЙ WINDOWS СЕРВЕР
Архитектура:
Пользователь → Веб-сайт → Центральный Windows Server → Outlook API
                              ↓
                    Черновик создаётся в Outlook сервера
                              ↓
                    Автопересылка на email пользователя
Как работает:

На одном Windows Server установлен Outlook
Flask сервер работает на этом сервере
Пользователь создаёт письмо на сайте
Запрос идёт на сервер → создаётся черновик в Outlook сервера
Сервер пересылает черновик на email пользователя
Пользователь открывает свой Outlook → видит письмо в папке "Входящие"

Технически:
python# На сервере:
def create_and_forward_draft(html, user_email):
    # 1. Создаём черновик в Outlook сервера
    outlook = win32.Dispatch("Outlook.Application")
    mail = outlook.CreateItem(0)
    mail.HTMLBody = html
    
    # 2. НЕ показываем, а СОХРАНЯЕМ и ОТПРАВЛЯЕМ
    mail.To = user_email
    mail.Subject = "[ЧЕРНОВИК] Ваше письмо"
    mail.Send()  # Отправляет пользователю
```

**Преимущества:**
- ✅ Один сервер для всех
- ✅ Пользователям ничего не нужно устанавливать
- ✅ Работает из любой точки

**Недостатки:**
- ❌ Письмо приходит как **входящее**, а не как черновик
- ❌ Нужен Windows Server + Outlook лицензия (~$50-100/мес)
- ❌ Пользователь получает письмо "от сервера", а не создаёт сам

### 🌐 ВАРИАНТ 2: BROWSER EXTENSION (Расширение для браузера)

**Архитектура:**
```
Пользователь → Веб-сайт → Расширение Chrome/Edge → COM API → Outlook.exe
Как работает:

Создаёте расширение для Chrome/Edge
Пользователь один раз устанавливает расширение
На сайте: нажимает "Создать письмо" → расширение перехватывает
Расширение использует Native Messaging → запускает .exe помощник
.exe помощник → вызывает Outlook COM API локально
Outlook открывается с письмом!

Технически:
Расширение (JavaScript):
javascript// background.js
chrome.runtime.onMessage.addListener((request, sender, sendResponse) => {
    if (request.action === 'createOutlook') {
        // Отправляем нативному помощнику
        chrome.runtime.sendNativeMessage(
            'com.yourcompany.outlookhelper',
            { html: request.html },
            response => sendResponse(response)
        );
    }
    return true;
});
Native Helper (.exe на ПК):
python# outlook_helper.py (скомпилирован в .exe)
import sys
import json
import win32com.client

def create_outlook_draft(html):
    outlook = win32com.client.Dispatch("Outlook.Application")
    mail = outlook.CreateItem(0)
    mail.HTMLBody = html
    mail.Display()

if __name__ == '__main__':
    message = json.loads(sys.stdin.read())
    create_outlook_draft(message['html'])
```

**Преимущества:**
- ✅ Устанавливается **один раз**
- ✅ Работает из браузера
- ✅ Outlook открывается **локально** у пользователя
- ✅ Не нужен постоянный локальный сервер

**Недостатки:**
- ❌ Нужно установить расширение (но это проще чем сервер)
- ❌ Нужен .exe помощник (~1MB, устанавливается с расширением)
- ❌ Только Chrome/Edge (Firefox сложнее)

**Подходит?** 🤔

### 📧 ВАРИАНТ 3: MAILTO + EML FILE (Хак)

**Архитектура:**
```
Пользователь → Веб-сайт → Генерирует .eml файл → Скачивает → Открывает
                                                          ↓
                                                    Outlook.exe
Как работает:

Генерируете .eml файл (стандартный email формат)
Пользователь скачивает файл
Двойной клик → Windows открывает в Outlook по умолчанию
Письмо открывается как черновик!

Технически:
javascriptfunction downloadEML(html, subject) {
    // EML формат
    const eml = `From: <user@example.com>
To: <recipient@example.com>
Subject: ${subject}
MIME-Version: 1.0
Content-Type: text/html; charset=UTF-8
Content-Transfer-Encoding: 8bit

${html}`;

    // Скачиваем файл
    const blob = new Blob([eml], { type: 'message/rfc822' });
    const url = URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = 'письмо.eml';
    a.click();
}
```

**Преимущества:**
- ✅ Не нужно НИЧЕГО устанавливать
- ✅ Работает сразу
- ✅ Outlook откроется автоматически (если по умолчанию)
- ✅ Самое простое решение

**Недостатки:**
- ❌ Нужно скачать файл (лишний шаг)
- ❌ Картинки должны быть base64 (размер файла)
- ❌ Не так элегантно

**Подходит?** 🤔

### 🔗 ВАРИАНТ 4: CUSTOM PROTOCOL HANDLER

**Архитектура:**
```
Пользователь → Веб-сайт → Кнопка "outlook://create?html=..." → Windows → Outlook.exe
Как работает:

Регистрируете custom protocol outlook:// в Windows Registry
Protocol связан с .exe помощником
На сайте: ссылка outlook://create?data=base64
Windows запускает ваш .exe
.exe создаёт письмо в Outlook

Технически:
Registry (устанавливается один раз):
regWindows Registry Editor Version 5.00

[HKEY_CLASSES_ROOT\outlook]
@="URL:Outlook Protocol"
"URL Protocol"=""

[HKEY_CLASSES_ROOT\outlook\shell\open\command]
@="\"C:\\Program Files\\OutlookHelper\\outlook-helper.exe\" \"%1\""
На сайте:
javascriptfunction createOutlookDraft(html) {
    const data = btoa(html); // base64
    window.location.href = `outlook://create?html=${data}`;
}
```

**Преимущества:**
- ✅ Элегантное решение
- ✅ Устанавливается **один раз**
- ✅ Работает как нативное приложение

**Недостатки:**
- ❌ Нужна установка .exe + регистрация protocol
- ❌ Ограничение на длину URL (~2000 символов)

**Подходит?** 🤔
```

# Email Builder -- Быстрый старт

## Windows

### 1. Разместить файлы

Положить рядом в одну папку:

```text
EmailBuilder.exe
config.ini
```

Для режима администратора -- дополнительно:

```text
.lic          файл с токеном (получить у ответственного за деплой)
```

### 2. Настроить config.ini

```ini
[app]
network_path_win = \\server\share\email-builder
port = 7788
```

### 3. Запустить

Дважды щёлкнуть `EmailBuilder.exe`.

При первом запуске приложение загрузит ресурсы из сетевой папки в локальный кеш.

---

## ALT Linux

### 1. Разместить файлы

Положить рядом в одну папку:

```text
EmailBuilder.sh
config.ini
```

Для режима администратора -- дополнительно:

```text
.lic          файл с токеном (положить рядом до запуска установщика)
```

### 2. Настроить config.ini

```ini
[app]
network_path_smb        = //server/share/email-builder
network_path_linux_hint = email-builder
port = 7788
```

### 3. Установить и запустить

```bash
bash EmailBuilder.sh
```

Приложение устанавливается в `~/EmailBuilder/`, на рабочем столе
создаётся ярлык.

### Обновление

Запустить новый `EmailBuilder.sh` той же командой. Установщик
автоматически определит установленную версию и при необходимости
обновит только бинарный файл. Настройки (`config.ini`) не затрагиваются.

---

## Если сетевая папка не найдена

При запуске появится диалог поиска:

- **Проверить путь** -- проверяет введённый путь вручную.
- **Выполнить поиск** -- автоматически сканирует подключённые сетевые ресурсы (или локальные диски, если выбран режим "Локально").
- **Создать новый** -- инициализирует пустой каталог по указанному пути (только администратор).

После успешного обнаружения путь сохраняется в `config.ini`.

---

## Режимы работы

### Пользователь

Открывается по умолчанию при отсутствии `.lic`.

- Создание и редактирование писем
- Применение шаблонов из общей библиотеки
- Отправка через Exchange

### Администратор

Открывается при наличии корректного `.lic` рядом с исполняемым файлом.

- Все возможности пользователя
- Управление общими шаблонами (создание, переименование, обновление, удаление)
- Управление категориями шаблонов
- Управление общими ресурсами (иконки, баннеры и др.)
- Переключение в режим пользователя через меню

---

## Работа без сети

Если сетевая папка недоступна, приложение запустится из локального
кеша с предупреждением. Все ранее загруженные ресурсы и шаблоны доступны.

---

## Обновление ресурсов

Когда администратор публикует новую версию ресурсов на сервере
(обновляет `version.txt`), приложение при следующем запуске предложит
загрузить обновление.

---

## Настройки Exchange

При первом запуске потребуется ввести учётные данные Exchange:

- Сервер EWS (например, `https://mail.company.ru/EWS/Exchange.asmx`)
- Логин и пароль

Данные сохраняются зашифрованными локально.

---

## Диагностика

| Симптом | Действие |
| --- | --- |
| Приложение не запускается -- ошибка конфига | Проверить `config.ini`, убедиться что указан `network_path_win` (Windows) или `network_path_smb` (Linux) |
| После обновления запускается старая версия (Linux) | Убедиться что запускается именно новый `EmailBuilder.sh`, а не ярлык от старой версии |
| Предупреждение о недействительном токене | Файл `.lic` повреждён или не соответствует сборке -- обратиться к администратору |
| Письма не отправляются | Проверить настройки Exchange в приложении |
| Ресурсы не обновляются | Проверить доступность сетевой папки, запустить обновление вручную через меню |
